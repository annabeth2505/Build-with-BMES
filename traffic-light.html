Project 03 · The big one

A traffic light that listens.

Build a real working traffic light controlled by a button. Then unlock five bonus levels — each one adds a new feature: a yellow transition, a buzzer, an LCD screen, a live countdown, and finally a fully automated pedestrian system.

Base Level: Cycle three lights with a button

Three LEDs (red, yellow, green) and one button. Each press of the button changes which light is on. Red → Green → Yellow → Red → … you choose the order. Finish this to unlock the bonus levels below.

Wire it up

  • Wire each LED to a different digital pin through its own 220Ω resistor — say pins 9, 10, 11
  • All LED short legs go to GND
  • Place the pushbutton across the gap in the middle of the breadboard so two of its legs are on each side
  • Connect one side of the button to a digital pin (try pin 8)
  • Connect the other side of the button to GND (this matters — see hint below)
03

Hints to write the base code

Use the Arduino's built-in pull-up. For the button pin, use pinMode(buttonPin, INPUT_PULLUP). This means the pin reads HIGH when the button is not pressed and LOW when it is pressed. (That's why the button connects to GND, not 5V.)
Track the current state. Make a variable like int lightState = 0; outside any function. 0 means red, 1 means green, 2 means yellow. Each press of the button advances the state.
Detect the press, not the hold. Read the button with digitalRead(buttonPin). Compare it to a lastButtonState variable — only do something when the button changes from HIGH to LOW. Otherwise it cycles through all three lights in 0.1 seconds while you're still holding it.
Cycle without overflowing. After advancing the state, use lightState = (lightState + 1) % 3;. The % 3 wraps it back to 0 after 2. This is one of the most useful tricks in programming.
Turn off everything before turning one on. Each time the state changes: turn off all three LEDs, then turn on the one that matches the new state. Otherwise lights stack on top of each other.

Bonus levels — unlock as you go

Each level adds one new feature on top of the previous one. Don't skip — they build on each other. Click "I did it!" when each level works to unlock the next.

1

Yellow as a transition

Add a 2-second pause
🔒 Locked

Right now the lights cycle instantly. Real traffic lights warn you before changing — yellow flashes for a moment before green takes over.

Goal: When the button is pressed and you're switching to green, light up yellow first for 2 seconds, then turn on green.

The simplest tool: delay() Inside the "switch to green" branch, turn yellow on, call delay(2000), turn yellow off, then turn green on. Order matters.
Watch out: delay() freezes everything — your button won't respond during those 2 seconds. That's fine for now. (Bonus 4 fixes this with a smarter approach.)
2

Pedestrian buzzer

Beeps when it's safe to cross
🔒 Locked

Real crosswalks beep so people who can't see the light know when to walk. Add a piezo buzzer that beeps three times when the green light turns on.

New component: 1× Piezo buzzer. Connect one leg to a digital pin (say pin 12), the other to GND.

Make sound with tone() tone(buzzerPin, frequency, duration) — for example tone(12, 1000, 200) plays 1000Hz for 200 milliseconds. Higher frequency = higher pitch.
Repeat with a loop. Use a for loop to beep three times. Inside the loop: tone(...), then delay(300) so the beeps don't overlap.
When should it beep? Only when green just turned on — not on every loop. Put the beep code inside the same branch where you turn green on.
3

"WALK NOW" on an LCD

Add a real screen
🔒 Locked

Add an LCD screen that shows what pedestrians should do: "STOP", "GET READY", or "WALK NOW" depending on which light is on.

New components:

  • 1× LCD 16x2 (the one in the Tinkercad components panel)
  • 1× Potentiometer (small dial) — controls how dark the text appears
  • ~10 more jumper wires
You'll need a library. Add #include <LiquidCrystal.h> at the top of your code. Then create the LCD object: LiquidCrystal lcd(7, 6, 5, 4, 3, 2); — those are the pin numbers it'll use.
Wire pattern (Tinkercad LCD pin labels): VCC → 5VGND → GNDV0 → potentiometer middle RS → pin 7RW → GNDE → pin 6 DB4 → 5DB5 → 4DB6 → 3DB7 → 2 LED+ → 5VLED- → GND
Important: the button pin moves. The LCD now uses pin 2. Move your button to pin 8 (or any free pin) and update your code.
Display text. In setup(), call lcd.begin(16, 2). To write something: lcd.clear(), then lcd.setCursor(0, 0), then lcd.print("WALK NOW"). Set cursor to (0, 1) for the second row.
If the screen is black squares: Turn the potentiometer dial. It controls contrast. The sweet spot is somewhere in the middle — when text appears clearly and the squares fade.
4

Live pedestrian countdown

10… 9… 8… on the LCD
🔒 Locked

When the green light is on, show a live 10-second countdown on the LCD. Each second, the number ticks down. Last 3 seconds, the buzzer beeps a warning.

Time without freezing: millis() millis() returns how many milliseconds the Arduino has been running. Save the value when green turns on: greenStart = millis();. Then in loop(), check the difference: millis() - greenStart tells you how much time has passed.
Convert to seconds and count down. int elapsed = (millis() - greenStart) / 1000; gives seconds passed. int remaining = 10 - elapsed; gives time left.
Update the LCD only when the number changes. If you redraw every loop, the screen flickers. Save the last shown number; only call lcd.print() when the new number is different.
Beep when it's almost up. If remaining is 1, 2, or 3 — call tone(buzzerPin, 880, 150).
Why this beats delay(): With millis(), the button still works during the countdown. The Arduino keeps checking everything in parallel. This is how real-time systems work.
5

Auto-return to red

A fully automated traffic light
🔒 Locked

The grand finale. When the countdown hits zero, show "TIME IS UP" on the LCD for 1.5 seconds, then automatically switch back to red — no button press needed. Now you have a real traffic light: press the button to request a walk, get your countdown, get returned to stop. Hands-free.

When countdown reaches zero: Display "TIME IS UP" on the LCD. Use delay(1500) to let pedestrians read it.
Then transition back to red. Turn green off, turn red on. Update the LCD to whatever you show during red ("STOP" or similar).
Don't forget the state variable. Set lightState = 0; so the next button press still cycles correctly. If you skip this, the state machine gets out of sync and the lights go wrong on the next press.
Bonus polish: Reset the countdown variable too, so if the user presses again, you start at 10 instead of -3.

#include <LiquidCrystal.h>
LiquidCrystal lcd(7, 6, 5, 4, 3, 2);

// --- pin assignments ---
const int RED_PIN    = 11;
const int YELLOW_PIN = 10;
const int GREEN_PIN  =  9;
const int BTN_PIN    =  8;
const int BUZZER_PIN = 12;

// --- state tracker ---
int lightState   = 0;    // 0=red  1=green  2=yellow
int lastBtnState = HIGH; // HIGH = not pressed (INPUT_PULLUP)

// --- pedestrian countdown ---
unsigned long greenStartTime = 0;
bool greenActive  = false;
int  walkCountdown = 10;

void setup() {
  pinMode(RED_PIN,    OUTPUT);
  pinMode(YELLOW_PIN, OUTPUT);
  pinMode(GREEN_PIN,  OUTPUT);
  pinMode(BUZZER_PIN, OUTPUT);
  pinMode(BTN_PIN,    INPUT_PULLUP);

  lcd.begin(16, 2);

  digitalWrite(RED_PIN, HIGH);
  showRed();

  Serial.begin(9600);
}

void loop() {
  int btnState = digitalRead(BTN_PIN);

  // detect the MOMENT of button press
  if (btnState == LOW && lastBtnState == HIGH) {
    digitalWrite(RED_PIN,    LOW);
    digitalWrite(YELLOW_PIN, LOW);
    digitalWrite(GREEN_PIN,  LOW);
    greenActive = false;

    lightState = (lightState + 1) % 3;

    if (lightState == 0) {
      digitalWrite(RED_PIN, HIGH);
      showRed();
    }

    if (lightState == 1) {
      // yellow for 2 seconds to warn drivers
      digitalWrite(YELLOW_PIN, HIGH);
      showYellowWarning();
      delay(2000);
      digitalWrite(YELLOW_PIN, LOW);

      // now switch to green + start pedestrian timer
      digitalWrite(GREEN_PIN, HIGH);
      greenStartTime = millis();
      greenActive    = true;
      walkCountdown  = 10;
      showGreen(walkCountdown);
      pedestrianBeep();
    }

    if (lightState == 2) {
      digitalWrite(YELLOW_PIN, HIGH);
      showYellowWarning();
    }

    delay(50);
  }

  // live countdown while green is active
  if (greenActive) {
    int elapsed   = (millis() - greenStartTime) / 1000;
    int remaining = 10 - elapsed;

    if (remaining != walkCountdown && remaining > 0) {
      walkCountdown = remaining;
      showGreen(walkCountdown);
      if (remaining <= 3) {
        tone(BUZZER_PIN, 880, 150); // urgent beep
      }
    }

    if (remaining <= 0) {
      greenActive = false;
      showWait();
      delay(1500);

      digitalWrite(GREEN_PIN, LOW);
      digitalWrite(RED_PIN, HIGH);
      lightState = 0;
      showRed();
    }
  }

  lastBtnState = btnState;
}

void showRed() {
  lcd.clear();
  lcd.setCursor(0, 0); lcd.print("     STOP       ");
  lcd.setCursor(0, 1); lcd.print(" Wait for green ");
  Serial.println("Now: RED");
}

void showYellowWarning() {
  lcd.clear();
  lcd.setCursor(0, 0); lcd.print(" PREPARE TO GO  ");
  lcd.setCursor(0, 1); lcd.print("  Green coming! ");
  Serial.println("Now: YELLOW");
}

void showGreen(int seconds) {
  lcd.clear();
  lcd.setCursor(0, 0); lcd.print("  WALK NOW :)   ");
  lcd.setCursor(0, 1); lcd.print("  Time left: ");
  lcd.print(seconds);
  lcd.print("s  ");
  Serial.print("GREEN - Countdown: "); Serial.println(seconds);
}

void showWait() {
  lcd.clear();
  lcd.setCursor(0, 0); lcd.print("  TIME IS UP!   ");
  lcd.setCursor(0, 1); lcd.print(" Wait for signal");
  Serial.println("Countdown ended");
}

void pedestrianBeep() {
  // 3 quick beeps = crossing signal starting
  for (int i = 0; i < 3; i++) {
    tone(BUZZER_PIN, 1000, 200);
    delay(300);
  }
}

Compare your version to this one. What did you do differently? Did you use functions like showRed(), or did you write everything inline? Did you use millis() for the countdown, or delay()? Both can work — but only one keeps the button responsive.
