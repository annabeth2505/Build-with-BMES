Project 02

Mix colors with code.

Type three numbers — red, green, blue — into the Serial Monitor. Watch your LED change color in real time. You're now writing software that talks to hardware. That's the whole job.

Type a color, see a color

Open the Serial Monitor, type something like 255,0,128, and the RGB LED should glow that exact color. Numbers from 0–255 for each channel.

The RGB LED has 4 legs Three are for red, green, and blue. The longest leg is the common — connect it to GND. Hover over the LED in Tinkercad to see which leg is which.
02

Wire it up

  • Place the RGB LED on the breadboard so each leg is in its own row
  • Connect the long (common) leg to GND
  • Run a 220Ω resistor from each of the other 3 legs to a digital pin that supports PWM — the ones marked with a ~ symbol (try pins 9, 10, 11)
  • Keep track of which pin connects to which color — you'll need that in your code
Why PWM pins (the ones with ~)? Regular digital pins only know ON or OFF. PWM pins can fake "in-between" brightness by turning on and off really fast. That's how you get 256 shades per color instead of just two.
03

Hints to write the code

Set up three output pins. In setup(), declare each color pin as an OUTPUT — same as in the blink project, but three of them.
Open the channel between Arduino and your computer. Add Serial.begin(9600) in setup(). This lets the Arduino send and receive text through the Serial Monitor.
Brightness ≠ digitalWrite. For variable brightness, use analogWrite(pin, value) where value is between 0 (off) and 255 (full blast). This only works on PWM pins (~).
Reading what someone types. In loop(), check Serial.available(). If there's data waiting, read three numbers using Serial.parseInt(). It automatically pulls out the next integer from the input.
Print it back to confirm. After reading the values, use Serial.print() and Serial.println() to echo them back so you can see what the Arduino received. Great for debugging.
Format you'll type. Type three numbers separated by commas or spaces, like 200,50,255 and press Enter. parseInt handles the separators for you.
04

Try these once it works

  • Type 255,0,0 for pure red. Then 0,255,0. Then 0,0,255.
  • What numbers make purple? Yellow? White? Hot pink?
  • Make the LED slowly fade between two colors using a for loop and small delays
  • Print a "color name guesser" message: if R is highest, print "Looks reddish!"

PWM pins only

Use pins with ~Pins 3, 5, 6, 9, 10, 11 support analogWrite. Others won't fade — they'll only do full on/off.

Code cheat sheet

Brightness, not just on/offanalogWrite(pin, value) — value is 0–255.
Open the Serial channelSerial.begin(9600) in setup(). Then click "Serial Monitor" inside the Tinkercad code editor.
Wait for inputif (Serial.available()) { … } only runs when there's something to read.
Read three numbersint r = Serial.parseInt();
It auto-skips commas, spaces, and newlines.
Echo it backSerial.print("R = "); + Serial.println(r);. Confirms what was received.
